## 技术架构优化

### 当前架构

- 纯静态网页，可部署在 GitHub Pages。
- 题库数据保存在 `data/story-items.json` 和 `data/opinion-items.json`。
- 图片资源保存在 `assets/story-images/`。
- 练习记录当前保存在浏览器 `localStorage`。
- 录音/录像当前保存在浏览器 `IndexedDB`。

### 短期技术优化

- [ ] 统一本地数据层
  - 将练习记录从 `localStorage` 迁移到 `IndexedDB`。
  - 用同一套数据访问方法管理文字稿、评分、备注、媒体索引。
  - 避免未来记录增多后 `localStorage` 容量和性能受限。

- [ ] 数据结构版本管理
  - 为本地数据增加 `schemaVersion`。
  - 后续字段变化时支持自动迁移旧数据。
  - 避免更新应用后历史记录无法读取。

- [ ] 完整导入/导出能力
  - 支持导出所有练习记录、评分、备注、文字稿。
  - 支持导入备份文件恢复数据。
  - 录音录像体积较大，可先只导出文本和评分，媒体备份作为后续增强。

- [ ] 本地存储健康提示
  - 在设置或记录页提示数据保存在当前浏览器本地。
  - 提醒用户定期导出备份。
  - 如果浏览器存储空间不足，给出友好提示。

### 中期架构选项

- [ ] 评估云端同步方案
  - 当需要多设备同步、云端备份、老师查看记录时，再引入后端数据库。
  - 候选方案：Supabase、Firebase。
  - 保留静态题库，优先只同步练习记录和用户数据。

- [ ] 账号与权限设计
  - 如果引入云端同步，需要区分孩子、家长、老师角色。
  - 家长可以查看和编辑记录；孩子主要负责练习；老师可选只读或点评。

- [ ] 云端媒体存储评估
  - 录音录像文件较大，不适合直接放数据库。
  - 后续可使用 Supabase Storage、Firebase Storage 或其他对象存储。
  - 需要明确隐私策略和删除机制。

### AI 能力相关架构

- [ ] 语音转文字服务端化
  - 自动转写需要调用语音识别服务，不应把 API Key 放在浏览器前端。
  - 后续应增加轻量后端或 serverless function 作为代理。

- [ ] AI 生成参考稿与反馈
  - 逐题参考故事、观点理由、练习反馈可以接入 AI。
  - 建议先生成静态参考稿并人工 review，再放入题库。
  - 实时 AI 反馈需要后端保护密钥和控制成本。

### 架构演进建议

1. 第一阶段：继续静态站点，统一 IndexedDB 数据层，补导入/导出。
2. 第二阶段：增加可选云端同步，只同步练习记录和文字数据。
3. 第三阶段：增加 AI 转写、AI 反馈、逐题参考稿生成，并通过后端保护 API Key。
